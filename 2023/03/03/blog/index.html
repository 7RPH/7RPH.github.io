由于笔者对香港轻量级服务器的态度从[每月花24rmb喝杯奶茶不如租台服务器]变成了[租台服务器不如喝杯奶茶]，就准备把hk的服务器关停(当然其实有其他不可抗力在里面)，这样一来之前在hk服务器上搭的博客也就得进行迁移了，考虑再三，由于笔者实在是不想走大陆服务器认证的繁冗流程，也没有钞能力去维护服务器的开销，只好再次回头捡起闲置许久的hexo配合github page搭一个轻量级的博客了。由于我与hexo也算是阔别已久，加之常用pc换成了mac，之前的环境全都没了，既然要重新搭建，那就记录一下过程做为第一篇文章好了。

本地环境部署

由于mac环境之前已经装好了nodejs跟npm，不再赘述，这里从hexo开始说起

hexo

为了方便的使用hexo，首先需要安装他的CLI(command-line interface)，由于前面已经装好了npm，直接下载

基本环境搭建好后，我们就用hexo新建一个文件夹来做为博客的根目录

该命令执行完毕后会新建一个名为blog的文件夹并在其中部署诸多环境，tree命令结果大概如下

这样环境初始化完成，可以顺便展示一下hexo的命令

值得注意的是，hexo的命令允许缩写，即hexo generate和hexo g是一个东西，都生成了静态文件，也就是我们最终的博客站点的文件

这里不妨hexo g看看生成结果

这里生成了一堆静态网页的资源文件，如果是第一次执行hexo g，那么还会先新建一个叫做public的文件夹，这些生成的文件都会放在public文件夹下，熟悉web的可能都知道，我们有了静态文件，那么我们直接起一个server就可以访问网页了，根据hexo提供的方法，我们执行hexo s开启服务器，会返回如下结果

这个命令是把public内的文件作为网站根目录并启动服务器，访问http://localhost:4000/即可看到我们的博客

皮肤

初始的博客并不是很符合我们的审美，得想个办法换一下，这里我选的是redefine主题，文档在此处https://redefine-docs.ohevan.com/

设置就根据喜好跟着文档随便改了改，本来就是为了方便，比较方便的设置配置文件的方法是去node_modules->hexo-theme-redefine下找到_config.yml配置文件把他粘贴到blog目录的_config.redefine.yml文件中然后对应修改

但是总不能一直用localhost孤芳自赏吧，所以我们要把hexo生成的静态文件上传到公网上，这就要用到github page了

github page

github page的原理简单理解就是会把你放在repo里的html，css，js等文件，放在服务器上运行，所以在这里我们就需要把public路径下的文件上传到github做为一个repo，而笔者作为github的苦手，这里就用github desktop偷懒了

域名绑定

想用个性点的域名而不是枯燥的github.io，使用github提供的custom domain功能把自己购买的域名绑定过去，具体流程可以看https://docs.github.com/en/pages/configuring-a-custom-domain-for-your-github-pages-site
